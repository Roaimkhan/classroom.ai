OUTPUT RULES (NON-NEGOTIABLE):

JSON SCHEMA:
{
  "status": "success | partial | error",
  "assumptions": ["string"],
  "answers": [
    {
      "question_id": "string",
      "response": "string",
      "steps": ["string"],
      "confidence": 0-1
    }
  ],
  "errors": ["string"]
}

FIELD DEFINITIONS:
- status:
  - "success" → all tasks completed correctly
  - "partial" → incomplete due to ambiguity or missing data
  - "error" → cannot proceed

- assumptions:
  - Explicit list of inferred or missing data used

- answers:
  - One entry per question/task
  - question_id → unique identifier (e.g., Q1, part_a)
  - response → final answer only
  - steps → minimal reasoning steps (concise)
  - confidence → float between 0 and 1

- errors:
  - Empty if none
  - Otherwise list encountered issues

FAILURE HANDLING:
- If any requirement cannot be satisfied → still return valid JSON
- Never break schema for any reason

- Ensure correct data types

REGENERATION RULE:
